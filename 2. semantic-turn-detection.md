## Testing mindset
- So agent deals with messy real world inputs, People mumble. They change their minds, midstence
and they asks things you never anticipted. So start to think how you'll test these scenarios
now will save your headaches later

- So you have to think about edge cases, different accents, background noises and interruptions.
- So agent relies only on voice activity detection it hears a pause and it thinks we're done. But humans pause
mid-setence all the time. We think we breathe we change direction, So a good conversation agent needs to understand when someone is actually finished speaking

- So turn detection decides when the agent should speak and when it should stay quiet and without it you would get interruptions and agent start answering at every single pause. Even if the human is still mid-sentence.


## VAD vs Semantic Detection
- VAD detects human patterned audio but humans pause, restart sentences, and trail off all the time..
so VAD sees a pause and it thinks that you're done talking. 
- But Semantic detection looks at all of the meaning of the sentence and it asks does this feel like a complete thought? The most important job of a turn detector is to reduce unwanted interruptions and it also improves STT engine accuracy since it gets complete utterneces instead of short spoken sentence and latency impacts is minimal only around 20ms for the agent to reply due to turn detection.
  - Reduces unwanted interruptions
  - Improves STT accuracy.
  - ~20ms latency impact.
