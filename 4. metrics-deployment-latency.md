## Metrics, deployment and latency optimization
- TTFA (Time to first audio, when the user stops speaking to when they hear the first word of the response from agent)
- TTFT (Time to first token, When the LLM starts generating which helps isolate wheather delays are coming from transcription the model or speech synthesis.)
- Token Usage (You'll also track token usage for cost estimation)
- Interruption rate (Interruption rate to see how often users cut off the agent)
- Fallback activation (Fallback activation if our primary providers fail)

# TTFA 
#### We want TTFA to be less than 1s for most responses
- Agent cycles through several states during a conversation listening, thinking and speaking the agent state changed event. It fires each time the agent transition between these states.

### Preemitive generation
- Preemtive generation helps the agent start forming a response before the user finishes speaking.
- The agents wait for a clear end of turn before actually speaking, but the thinking has already begun
- This way save hundreds of milliseconds off of perceived latency especially for longer user turns
- But premitive generation is not good all the time, There are few scenarios where this trade-off might not work for your favor, for e.g -> Mid sentence direction changes 
- Suppose user says book me a flight for new york ..... no book me a flight for chicago, The LLM was already commited to new york
- Multi part instruction, long requests with multiple steps can get partially answered before the full context is available.
- High Accuracy domain (Medical, Legal, Financial application) where it getting it right matters more than being fast anything, there we should disabled it or tuning turn detection threshold.
