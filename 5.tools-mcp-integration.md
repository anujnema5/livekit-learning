# Tools and MCP integrations
- Tools let your agent call external APIs, lookup data and take actions on behalf of users.
- WIthout tool agent can only responds on what on your training data and conversation context and that's pretty limiting.
- Tools let you fetch real time information like weather, stock prices or order status.
- They let you take action in external systems like creating tickets, sending emails or updating records.
- You can access your private data from customer databases, updating records getting data from internal docs.
- And you can trigger UI updates on the frontend via RPC tools

## So livekit uses the function tool decorator to expose python functions to the LLM. The LLM decides when to call them based on conversation context.

## Model Context Protocol (MCP)
MCP (Model Context Protocol) is an open-source standard for connecting AI applications to external systems.

Using MCP, AI applications like Claude or ChatGPT can connect to data sources (e.g. local files, databases), tools (e.g. search engines, calculators) and workflows (e.g. specialized prompts)—enabling them to access key information and perform tasks.

- In LiveKit MCP and local tools are treated same so when agent gets any query for anything it searches for the right tool available in the code.

### Tool Best practice:
- Be specific in descriptions.
- Keep tools fast (<2seconds)
- Handle errors with ToolError
- Return Meaningful data
### For long running tool we can use context.session.say("Let me search for that") or something

### So if we are using any task that should not be undone in between, like payment or booking, we can use context.session.disallow_iterruptions() to make sure it runs without interruption.
